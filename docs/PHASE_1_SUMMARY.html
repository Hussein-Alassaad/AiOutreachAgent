Nexaris · AI Outreach Agent

Phase 1 — Database Setup

A summary of the work completed
24 July 2026

What this phase covered

This phase focused on building the database that the entire outreach system runs on. Every lead the system will ever find, every message it writes, every account it uses, and every setting that controls its behaviour needed a proper home — one that is organised, secure, and reliable from day one.

Hussein set up a new database project and designed nine separate tables inside it, each responsible for one part of the system: the leads themselves, the messages sent to them, the record of every stage a lead moves through (from first contact to a closed deal), the three outreach accounts and their settings, the follow-up schedule, a permanent history of every business ever analysed, a log of every alert sent, and a record of each day's run. Together these nine tables form the complete backbone the rest of the project will be built on top of.

Keeping it secure

Because this system will eventually hold real business data and real conversations, Hussein put a security layer directly on the database itself, not just in the app that sits on top of it. Every table was locked down so that only approved, signed-in accounts can view or change anything inside it — nobody else, even with a copy of the connection details, can read or edit the data. This was verified table by table to make sure nothing was left unprotected.

Connecting the system to the database

With the database in place, Hussein built the layer of code that lets the rest of the program actually talk to it — reading leads, saving new ones, updating a message's status, moving a deal from one stage to the next, and so on. This was organised into one clear, well-structured module so that every future part of the system uses the same reliable, consistent way of reaching the data, rather than each piece doing its own thing.

Testing it for real

Rather than assuming everything worked, Hussein ran a full live test directly against the real database: confirming every one of the nine tables exists and is reachable, confirming the starting data (the three outreach accounts and the default settings) was correctly in place, and finally adding a test record, reading it back to make sure it matched exactly, and removing it again so the database was left perfectly clean afterward.

Result: every check passed. The database is live, secured, fully connected, and confirmed working end to end.
